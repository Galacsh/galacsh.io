SEO를 위해 필요한 핵심 작업
search-enginesitemaprobots

트래픽을 올리기 위한 방법은 다양하겠지만, 그 중에 하나는 연관 검색어가 검색되었을 때 자신의 웹 사이트가 자주, 상단에 표시되게끔 하는 것이 있을 것입니다.

검색 결과에 자주 포함되고, 상단에 노출되게끔 하는 일련의 작업들을 검색 엔진 최적화 (SEO, Search Engine Optimization) 작업이라고 할 수 있겠습니다.

검색 결과는 검색 엔진이 여러 웹 사이트를 검토하여 적합한 결과를 표시하는 것이므로, 검색 엔진이 웹 사이트를 잘 검토할 수 있도록 작업해두는 것이 기본이 되어야 합니다. 이 기본 작업이 되어 있어야 검색어에 맞는 컨텐츠들이 빛을 보겠죠?

검색 엔진 최적화를 위해 필요한 작업들을 설명하기에 앞서 검색 엔진이 동작하는 방식을 알아야 작업들의 목적을 조금 더 명확히 알 수 있으므로, 관련하여 간단히 정리한 글을 확인해주세요.

오늘은 이러한 기본 (핵심) 작업들에 대해서 알아보겠습니다.

검색 엔진에 웹 사이트 등록

우선 검색 엔진이 웹 사이트가 존재하는지 알아야 검토를 하든 할테니 가장 먼저 검색 엔진에 웹 사이트를 등록하는 작업이 필요합니다.

관련해서는 이 글을 참고해주세요.

sitemap.xml

Next.js 사용 시 참고: Metadata Files: sitemap.xml | Next.js (nextjs.org)

웹 사이트의 존재 여부를 검색 엔진이 알게 되었다고 하더라도, 사이트 내의 모든 페이지를 알기는 어렵습니다. 따라서 검색 엔진의 크롤러가 웹 사이트 내 페이지들을 인지할 수 있도록 정보를 주어야 합니다.

즉, sitemap.xml은 웹 사이트 내 페이지 목록 정보를 담은 파일이라고 보면 되겠습니다. 일반적으로 sitemap.xml 파일은 https://example.com/sitemap.xml 처럼 최상위 경로에서 제공하도록 약속되어 있습니다.

이 XML 파일은 Sitemap 프로토콜을 따르는 XML 파일입니다. Sitemap 프로토콜은 여기에서 확인하실 수 있습니다.

주요 내용만 뽑아보자면,

파일의 형태는 다음과 같으며:

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
    <url>
        <loc>http://www.example.com/</loc>
        <lastmod>2005-01-01</lastmod>
        <changefreq>monthly</changefreq>
        <priority>0.8</priority>
    </url>
    <url>
        <loc>http://www.example.com/1234</loc>
    </url>
    ...
</urlset>

각 항목에 대한 설명은 다음과 같습니다.

  • 필수
    • urlset: 전체를 감싸는 역할
    • url: 하나의 URL을 감싸는 역할
    • loc
      • 페이지의 URL
      • 프로토콜(예: http)로 시작해야 함
      • 2,048자 미만이어야 함
  • 선택사항
    • lastmod
      • 페이지를 마지막으로 수정한 날짜
      • W3C Datetime 포멧을 따라야 함
      • YYYY-MM-DD 형식도 가능함
    • changefreq
      • 페이지 내용의 변경이 얼마나 자주 일어나는지 표기
      • 다만, 크롤링 주기를 보장하지는 않음 (검색 엔진 재량임)
      • 가능한 값
        • always, hourly, daily, weekly, monthly, yearly, never
    • priority
      • 사이트의 다른 URL에 비한 이 URL의 우선순위
      • 가능한 값: 0.0 ~ 1.0 (생략 시 0.5)
      • 검색 결과에 영향을 미치지는 않음
      • 단순히, 크롤러에게 어떤 페이지가 정말 중요한지 알려주는 용도

robots.txt

Next.js 사용 시 참고: Metadata Files: robots.txt | Next.js (nextjs.org)

검색 엔진의 크롤러는 Sitemap 외에도 수집된 페이지 속 링크 등을 이용해 페이지들을 수집합니다. 그 페이지 중에 크롤링 되지 않았으면 하는 페이지들은 어떻게 할까요? 크롤러에게 해당 경로가 접근 가능한지 알려주면 됩니다.

크롤링 허용 여부 정보를 모아두는 파일이 robots.txt 입니다. 일반적으로 robots.txt 파일은 https://example.com/robots.txt 처럼 최상위 경로에서 제공하도록 약속되어 있습니다.

표준 프로토콜은 여기에서 확이할 수 있습니다. Google 검색 센터에서 간략하게 작성한 글도 있으니 참고하세요.

복잡하게 크롤러의 접근을 막을 페이지가 있는 것이 아니라면, 다음 예제 중에 골라 사용해도 충분합니다. (Sitemap 항목은 선택사항입니다.)

예시 1) 전체 허용

User-agent: *
Allow: /

Sitemap: https://example.com/sitemap.xml

예시 2) 전체 비허용

User-agent: *
Disallow: /

Sitemap: https://example.com/sitemap.xml

웹 사이트 신경쓰기

검색 엔진이 웹 사이트를 인지했다면, 이제는 웹 사이트를 파악하고 분석할 차례입니다.

탐색하기 좋은 URL

Google 검색 센터에서는 디렉토리 형식으로 사이트 구조를 계층화하라고 합니다. 설명 URL, Breadcrumb 등으로 표현되어 있는 개념인데, 쉽게 말하면, URL 구조만으로 를 이해하고 탐색하기 좋게 하라는 내용입니다.

  • 나쁨
    • https://www.example.com/2/6772756D707920636174
  • 좋음
    • https://www.example.com/pets/cats.html

관련 있는 내용들끼리 같은 디렉토리로 묶는 것도 포함합니다.

HTML로 의미 전달하기

  • 시각적 컨텐츠의 텍스트
    • 이미지나 비디오 등을 제공할 때 텍스트 정보를 제공해야 합니다.
    • e.g. <img alt='옆에서 바라본 소금빵' ...>
  • 제목과 설명
    • 메타 태그로 제공하는 제목과 설명이 필요합니다.
    • <title>검색 엔진 최적화</title>
    • <meta name="description" content="검색 엔진 최적화에 대해 ...">
  • Semantic HTML
    • 가능하다면 HTML 마크업을 사용합니다.
    • 또한 의미와 용도에 맞는 태그를 사용합니다.
    • CSS 속성을 이용하여 화면에 텍스트를 표현하는 방식은 지양합니다.
      • Google 검색에서는 이를 무시합니다.

페이지 경험 개선

모바일 친화성

Google에서는 색인을 생성하고 순위를 지정하는 데 모바일 버전의 콘텐츠를 우선적으로 사용한다고 합니다. 따라서 모바일 친화성도 신경써야 하는 부분입니다.

스마트폰으로 검색하는 것이 더 익숙하고 흔한 요즘, 사용성을 위해서라도 필요한 부분입니다. 자세한 내용은 이 글을 참고하세요.

로딩/렌더링 속도 개선

페이지 로딩은 물론 렌더링이 느리지 않은지 확인하고 개선해야 합니다.

HTTPS

주소창 옆에 자물쇠를 보여주는 HTTPS도 중요합니다. 크롤링 및 색인 생성 시 권장사항이기도 한 HTTPS는 사용자의 보안을 위해서도 필요합니다. 더 많은 내용은 이 글을 참고하세요.
